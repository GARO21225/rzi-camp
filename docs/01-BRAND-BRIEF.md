# Brand Brief — RZI Camp ERP

## 1. Qu'est-ce que c'est

RZI Camp est un ERP de gestion de camp minier résidentiel, développé pour le camp Roxgold Sango (Côte d'Ivoire) et conçu pour être réutilisable sur d'autres sites industriels/miniers en Afrique de l'Ouest.

**Promesse centrale** : remplacer les tableurs Excel et les remontées papier par un système temps réel unique, qui donne à un directeur de camp ou un responsable logistique la même sensation de contrôle qu'un centre d'opérations aéroportuaire — sans la facture à plusieurs centaines de milliers de dollars des plateformes type Palantir.

## 2. Public cible

| Profil | Besoin principal |
|---|---|
| Direction du camp / Admin | Vue d'ensemble, rapports, KPIs, validation des inductions |
| Agent logistique / Rotations | Planifier les convois, gérer les sièges, suivre qui est où |
| Technicien maintenance | Déclarer, suivre, clôturer les incidents |
| Équipe restauration / boutique | Gérer le stock, encaisser, suivre la consommation |
| Personnel résident | S'induire au camp, réserver un voyage, consulter ses infos |

## 3. Personnalité de marque

- **Robuste, pas léché** — priorité à la fiabilité opérationnelle sur l'esthétique pure. Le système doit fonctionner avec une connexion 3G en brousse.
- **Direct** — pas de jargon SaaS. "Déclarer un incident", pas "Créer un ticket".
- **Ambitieux dans les bons modules** — le Mission Control des rotations et l'Induction Camp visent l'effet "WOW" volontairement, car ce sont les points de contact qui forgent la perception "système premium".
- **Sans fioritures dans les modules utilitaires** — Maintenance, Boutique, Personnel restent des outils de travail, pas des vitrines.

## 5. Ce que RZI Camp n'est PAS

- Pas un ERP générique configurable — chaque module est taillé pour les opérations minières/camp.
- Pas un outil grand public — l'UX assume un utilisateur formé sur site.
- Pas dépendant d'une connexion stable — la robustesse réseau (mode offline, retry) est un critère de conception, pas une option.
